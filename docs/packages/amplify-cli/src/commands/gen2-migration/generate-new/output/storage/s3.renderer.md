# storage/s3.renderer.ts — S3Renderer

Pure renderer that produces TypeScript AST for `storage/resource.ts`.

## How It Works

`render(opts)` accepts a storage identifier, access patterns, triggers, and trigger function categories. It builds a `defineStorage()` call with:

- `name` — template literal using `branchName` (e.g., `myBucket-${branchName}`)
- `access` — arrow function with `allow.authenticated.to([...])`, `allow.guest.to([...])`, `allow.groups([...]).to([...])`, and `allow.resource(funcName).to([...])` patterns for public/_, protected/{entity_id}/_, and private/{entity_id}/\* paths
- `triggers` — `onUpload` and `onDelete` Lambda references

Function imports are added based on the function's category (contributed by `FunctionGenerator`) to resolve correct import paths (e.g., `../function/myFunc/resource` vs `../storage/myFunc/resource` for storage triggers).

## Relationship to Other Components

- Called by `S3Generator` — receives typed options, returns AST nodes
- No dependency on `Gen1App` — purely transforms input to AST
- Uses `renderResourceTsFile()` from `resource.ts` for the standard resource file structure
