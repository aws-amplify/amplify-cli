# storage/s3.generator.ts — S3Generator

Generates the S3 storage resource and contributes bucket overrides to backend.ts.

## How It Works

One per S3 bucket. In `plan()`, it reads the storage category from `Gen1App.meta`, fetches bucket config (notifications, acceleration, versioning, encryption) via `AwsFetcher`, and reads `cli-inputs.json` from the cloud backend for access patterns (auth, guest, groups, function access).

Renders `storage/resource.ts` via `S3Renderer` with the storage identifier, access patterns, triggers, and function category map. Then contributes to backend.ts:

- Import and `defineBackend` property for `storage`
- Bucket overrides: encryption config, bucket name comment for post-refactor

Trigger detection reads S3 notification configuration to find Lambda triggers (onUpload, onDelete).

## Relationship to Other Components

- Receives `Gen1App`, `BackendGenerator`, and `outputDir`
- Uses `S3Renderer` (pure) for resource.ts AST construction
- Exposes `addFunctionStorageAccess()` for `FunctionGenerator` to contribute S3 access permissions
- `DynamoDBGenerator` shares the same storage category but writes to backend.ts only (no resource.ts)
